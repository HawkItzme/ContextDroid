# v0.1.0-alpha.1

ContextDroid is an independently maintained Android-focused downstream of RTK. This first alpha
adds conservative Android Gradle, selected ADB, and bounded Logcat diagnostics with complete local
raw-output recovery, confidence-based fallback, omission accounting, and local quality proxies.

The default `contextdroid-safe` profile avoids automatic rewriting of pipelines, redirects,
structured/binary output, security tools, broad discovery commands, and unknown commands.
`android-only` limits automatic handling to verified Android families. `rtk-compatible` is an
explicit, more aggressive compatibility option.

Claude Code on Linux is the supported integration candidate. Cursor is experimental. Codex is
guidance-only and does not receive transparent shell interception. There is no `rtk` alias.

This is prerelease software. Diagnostic formats vary, device evidence is still expanding, and
many small outputs intentionally fall back to raw because semantic output would not be smaller.
See the benchmark report and validation evidence. Do not share unredacted raw logs.

Install without a local build using the checksum-verifying `install.sh` on Linux/macOS or
`install.ps1` on Windows. Every release archive is covered by `SHA256SUMS`, a machine-readable
release manifest, a CycloneDX SBOM, notices, and GitHub build provenance.
